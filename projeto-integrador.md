## 1. Apresentação Geral do Projeto

Este projeto consiste no desenvolvimento de um sistema web acadêmico, elaborado como parte do Projeto Integrador do Módulo D do Curso Técnico em Informática. O sistema simula um ambiente administrativo, permitindo a visualização e organização de informações por meio de páginas web estruturadas.

## 2. Objetivos

### 2.1 Objetivo Geral

Desenvolver um sistema web que simule um sistema administrativo, incluindo navegação, apresentação de dados e simulação de integração com banco de dados.

### 2.2 Objetivos Específicos

* Aplicar conceitos de desenvolvimento web.
* Utilizar HTML5 para estruturação das páginas.
* Utilizar CSS3 para estilização.
* Simular o uso de banco de dados para organização das informações.
* Consolidar os conhecimentos adquiridos no módulo.

## 3. Regras de Negócio

* **RN01** – O sistema deve permitir navegação entre páginas administrativas.
* **RN02** – O sistema deve manter padrão visual em todas as páginas.
* **RN03** – O sistema deve exibir informações simuladas de forma organizada.
* **RN04** – O sistema deve simular cadastro e listagem de dados.
* **RN05** – O sistema deve garantir consistência visual dos dados exibidos.

## 4. Requisitos Funcionais

* **RF01** – O sistema deve possuir página inicial.
* **RF02** – O sistema deve permitir acesso às páginas administrativas.
* **RF03** – O sistema deve exibir dados simulados em formato de tabela.
* **RF04** – O sistema deve simular consultas a banco de dados.
* **RF05** – O sistema deve permitir navegação por menu.

## 5. Requisitos Não Funcionais

* **RNF01** – Compatível com navegadores modernos.
* **RNF02** – Interface intuitiva.
* **RNF03** – Código organizado.
* **RNF04** – Baixo tempo de carregamento.

## 7. Modelo Conceitual de Banco de Dados

O modelo conceitual do sistema é composto pelas entidades **Usuário** e **Registro**, relacionadas para simular a organização e consulta de informações.

## 8. Descrição do Caso de Uso

### UC01 – Visualizar Dados

**Ator:** Usuário
**Objetivo:** Visualizar informações simuladas no sistema.

**Fluxo Principal:**

1. Usuário acessa o sistema.
2. Usuário navega até a página administrativa.
3. Sistema exibe dados simulados.

**Pós-condição:**
Dados exibidos corretamente.
